【准备阶段】
□ 已安装 Git
□ 拥有 GitHub 账号
□ 已下载/创建项目文件夹

【配置阶段】
□ 运行 setup.bat 或手动修改 package.json 中的用户名
□ 确认 homepage 字段格式正确：
  "homepage": "https://你的用户名.github.io/fund-valuation-system"

【GitHub 仓库创建】
□ 登录 GitHub
□ 创建新仓库，名称：fund-valuation-system
□ 选择 Public（公开）
□ 不勾选任何初始化选项
□ 复制仓库 URL

【代码上传】
□ 打开命令行，进入项目文件夹
□ 执行：git init
□ 执行：git add .
□ 执行：git commit -m "Initial commit"
□ 执行：git branch -M main
□ 执行：git remote add origin [你的仓库URL]
□ 执行：git push -u origin main
□ 确认代码已成功上传到 GitHub

【GitHub Actions 配置】
□ 进入仓库页面，点击 Actions 标签
□ 点击 "I understand my workflows, go ahead and enable them"
□ 左侧选择 "Auto Fund Valuation & Deploy"
□ 点击 "Run workflow" 按钮
□ 再次点击绿色的 "Run workflow" 确认
□ 等待运行完成（约 2-3 分钟）
□ 确认出现绿色对勾 ✓

【GitHub Pages 设置】
□ 进入 Settings → Pages
□ Source 选择 "gh-pages" 分支
□ 点击 Save
□ 记录显示的网站地址

【验证部署】
□ 等待 5-10 分钟（首次部署需要时间）
□ 访问网站：https://你的用户名.github.io/fund-valuation-system
□ 确认页面正常显示
□ 确认基金数据已加载
□ 测试刷新按钮

【功能测试】
□ 检查所有基金卡片是否正常显示
□ 检查估值数据是否正确
□ 检查重仓股信息是否显示
□ 测试响应式布局（手机端）

【自动化验证】
□ 查看 Actions 标签，确认定时任务已设置
□ 下次工作日 9:00-15:00 检查是否自动更新

【常见问题快速解决】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

问题：网站显示 404
解决：
  1. 确认 GitHub Pages 已开启
  2. 确认选择了 gh-pages 分支
  3. 等待 10 分钟后重试

问题：显示"暂无数据"
解决：
  1. 检查 Actions 是否运行成功
  2. 手动触发一次 workflow
  3. 查看 Actions 日志排查错误

问题：Actions 运行失败
解决：
  1. 点击失败的 workflow 查看详细日志
  2. 检查 Python 依赖是否正确
  3. 检查基金代码是否有效
  4. 重新运行 workflow

问题：样式显示异常
解决：
  1. 清除浏览器缓存
  2. 强制刷新（Ctrl + F5）
  3. 检查浏览器控制台错误
