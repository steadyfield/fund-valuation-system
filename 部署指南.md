# 🚀 GitHub Pages 部署完整指南

## 📋 前置准备
- 一个 GitHub 账号
- 本地已安装 Git

---

## 第一步：修改配置文件

### 1. 编辑 `package.json`
找到第 4 行的 `homepage` 字段，修改为你的实际信息：

```json
"homepage": "https://你的GitHub用户名.github.io/fund-valuation-system"
```

**示例**：如果你的 GitHub 用户名是 `zhangsan`，则改为：
```json
"homepage": "https://zhangsan.github.io/fund-valuation-system"
```

---

## 第二步：上传代码到 GitHub

### 1. 在 GitHub 网站创建新仓库
1. 登录 GitHub，点击右上角 `+` → `New repository`
2. **Repository name** 填写：`fund-valuation-system`
3. 选择 **Public**（必须是公开仓库才能用 GitHub Pages）
4. **不要**勾选 "Add a README file"
5. 点击 **Create repository**

### 2. 在本地上传代码
打开命令行（CMD 或 PowerShell），进入项目文件夹，执行：

```bash
git init
git add .
git commit -m "Initial commit: Fund valuation system"
git branch -M main
git remote add origin https://github.com/你的用户名/fund-valuation-system.git
git push -u origin main
```

**注意**：将 `你的用户名` 替换为你的实际 GitHub 用户名。

---

## 第三步：开启 GitHub Pages

1. 进入你的 GitHub 仓库页面
2. 点击顶部的 **Settings**（设置）
3. 在左侧菜单找到 **Pages**
4. 在 **Source** 下拉菜单中：
   - **暂时先不用设置**，等第一次 Actions 运行完会自动创建 `gh-pages` 分支
5. 保持页面打开，等待下一步

---

## 第四步：触发首次部署

### 1. 手动运行 GitHub Actions
1. 回到仓库首页，点击顶部的 **Actions** 标签
2. 如果看到提示 "Workflows aren't being run on this repository"，点击绿色按钮 **I understand my workflows, go ahead and enable them**
3. 在左侧找到 **Auto Fund Valuation & Deploy**
4. 点击右侧的 **Run workflow** 按钮
5. 在弹出的对话框中，点击绿色的 **Run workflow** 按钮

### 2. 等待部署完成
- 你会看到一个黄色圆圈在转动，表示正在运行
- 大约等待 **2-3 分钟**
- 当黄色圆圈变成 **绿色对勾 ✓** 时，表示部署成功

### 3. 配置 GitHub Pages（首次运行后）
1. 再次进入 **Settings** → **Pages**
2. 现在 **Source** 下拉菜单中应该可以选择 `gh-pages` 分支了
3. 选择 `gh-pages` 分支，点击 **Save**
4. 页面顶部会显示：**Your site is live at https://你的用户名.github.io/fund-valuation-system**

---

## 第五步：访问你的网站

打开浏览器，访问：
```
https://你的用户名.github.io/fund-valuation-system
```

🎉 **恭喜！你的基金估值网站已经上线了！**

## ❓ 常见问题

### Q1: 网站显示 404 Not Found
**解决方法**：
1. 确认 GitHub Pages 已经开启（Settings → Pages）
2. 确认选择了 `gh-pages` 分支
3. 等待 5-10 分钟，GitHub Pages 需要时间生效

### Q2: 网站显示"暂无数据"
**解决方法**：
1. 检查 Actions 是否运行成功（应该有绿色对勾）
2. 手动触发一次 workflow
3. 确认 `public/data/funds.json` 文件已生成

### Q3: Actions 运行失败
**解决方法**：
1. 点击失败的 workflow 查看错误日志
2. 常见原因：
   - Python 依赖安装失败 → 检查 `requirements.txt`
   - 基金代码错误 → 检查 `WATCH_LIST` 中的代码是否正确
   - 网络问题 → 重新运行 workflow

### Q4: 如何修改网站样式？
编辑 `src/index.css` 文件，修改颜色、字体等样式。
